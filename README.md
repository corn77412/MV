請幫我撰寫一個 **完整可執行的 Python 3.10+ 程式**，使用 **Pillow** 與 **moviepy** 製作「照片輪播音樂影片」。

## 功能需求
程式執行後，使用 `input()` 要求使用者依序輸入：
1. 照片資料夾路徑
2. 音樂檔路徑
3. 歌詞字幕檔路徑
4. 字型檔路徑（例如 `NotoSansTC-Black.ttf`）
5. 輸出影片檔路徑

程式需根據這些素材，自動產生一支 **1920x1080** 的音樂影片，並輸出為 MP4。

---

## 輸入檔案規格

### 照片
- 從使用者指定的資料夾讀取照片
- 支援格式：`.jpg`, `.jpeg`, `.png`, `.webp`
- 依檔名升冪排序播放
- 若資料夾不存在或沒有任何圖片，需顯示錯誤訊息並結束

### 音樂
- 使用者輸入音樂檔路徑
- 支援格式：`.mp3`, `.wav`, `.m4a`
- 最終影片總長度以音樂長度為準
- 若音樂檔不存在，需顯示錯誤訊息並結束

### 歌詞字幕
- 使用者輸入字幕檔路徑
- 使用 **LRC 格式**
- 程式一開始要提示使用者 LRC 格式範例，例如：
```txt
[00:00.00]第一句歌詞
[00:05.50]第二句歌詞
[00:10.00]第三句歌詞
```
- 每行格式為 `[分:秒.百分秒]歌詞內容`
- 每句歌詞從自己的時間點顯示，到下一句歌詞開始前結束
- 最後一句歌詞顯示到音樂結束
- 超出音樂長度的歌詞可忽略
- 若字幕格式錯誤，需顯示錯誤訊息並結束

### 字型
- 使用者輸入字型檔路徑
- 支援 `.ttf` 或 `.otf`
- 若字型檔不存在，需顯示錯誤訊息並結束

---

## 影片規格
- 解析度：`1920x1080`
- FPS：`30`
- 輸出格式：`MP4`
- 視訊編碼：`libx264`
- 音訊編碼：`aac`

---

## 照片播放規則
- 所有照片平均分配音樂總長度
- 每張照片顯示秒數 = 音樂長度 / 照片數量
- 最後一張照片結束時間需精準對齊音樂結束

---

## 畫面合成規則
每一張照片畫面都必須依照以下順序合成：
1. 建立 `1920x1080` 畫布
2. 先貼上模糊化背景照片
3. 再貼上原始照片
4. 最後貼上字幕

---

## 原始照片縮放規則
- 必須保持原始長寬比
- 原始照片需完整顯示於 `1920x1080` 畫面內
- 不可裁切
- 縮放比例使用 `min(1920/原寬, 1080/原高)`
- 縮放後置中顯示

---

## 模糊背景規則
- 以原始照片複製一份後，套用：
```python
img.filter(ImageFilter.BoxBlur(25))
```
- 模糊背景必須保持原始長寬比
- 模糊背景需等比縮放至可完整覆蓋 `1920x1080` 畫面
- 可超出畫面並裁切
- 縮放比例使用 `max(1920/原寬, 1080/原高)`
- 模糊背景置中後裁切為 `1920x1080`

---

## 字幕規則
- 使用使用者提供的字型檔
- 字體大小：`40`
- 字體顏色：白色
- 字幕水平置中
- 字幕垂直位置在畫面下方 1/5 處
- 請加上黑色描邊或陰影，避免白字看不清楚
- 若單行過長，可自動換行
- 字幕需依照時間正確切換
